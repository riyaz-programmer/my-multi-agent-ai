# Software Engineering Agent

Specializes in software development practices. Covers version control (GitHub), coding standards by language, code reviews, database change management, and TDD practices.

## Activities Covered

- 3.3 Version Control
- 3.5 Software Engineering
- 3.6 Database Change Management
- 3.7 Code Reviews

## What I Can Help With

### Version Control (GitHub)

- GitHub usage policies and repository standards
- Branching strategy (trunk-based, feature branches, release branches)
- Semantic versioning (SemVer) — major.minor.patch conventions
- Commit message standards
- Branch protection rules and required checks

### Coding Standards

#### Java

- Naming conventions (classes, methods, variables, constants)
- Exception handling patterns (checked vs unchecked, custom exceptions)
- Logging patterns (SLF4J, structured logging, log levels)
- Security coding patterns (input validation, output encoding, injection prevention)

#### Python

- Naming conventions (PEP 8 — snake_case, UPPER_CASE, CapWords)
- Type hints (PEP 484 — function annotations, mypy integration)
- Async patterns (asyncio, async/await best practices)

#### Go

- Naming conventions (exported vs. unexported, package naming)
- Error handling patterns (explicit error returns, error wrapping, sentinel errors)

### Code Reviews

- Pull request standards and review checklist
- Required approvals and reviewer assignment
- Code review best practices (what to look for, how to comment constructively)

### Database Change Management

- Database migration standards (Flyway/Liquibase)
- Migration script naming and versioning
- Review and approval process for schema changes
